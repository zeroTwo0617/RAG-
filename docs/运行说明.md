# 运行说明

## 前置依赖

- Docker（用于一键起 PostgreSQL + pgvector）
- JDK 17+（本工程编译用 Java 19 验证）
- Node.js 18+
- Maven 3.8+（或用后端自带的 `./mvnw`）

## 1. 启动数据库

```bash
docker compose up -d
```

镜像 `pgvector/pgvector:pg16` 已内置 `vector` 扩展，应用启动时 Flyway 会自动建表。

## 2. 启动后端

```bash
cd backend
cp .env.example .env        # 默认配置即可跑（Embedding 走本地兜底）
mvn spring-boot:run         # 或 ./mvnw spring-boot:run
```

- 接口基址：http://localhost:8080/api
- Swagger 文档：http://localhost:8080/swagger-ui.html
- 健康：curl http://localhost:8080/api/documents

### 关于 Embedding（重要）

默认 `embedding.provider=auto`：

- 若未配置 `EMBEDDING_API_KEY` → 自动使用**本地 hash 兜底**（离线、无需任何 key，向量语义质量较低，仅供 M1 演示）
- 若配置了 OpenAI 兼容的 key → 调用远程 Embedding（语义质量高）

要接真实语义模型，在 `.env` 设置：

```
EMBEDDING_PROVIDER=api
EMBEDDING_API_BASE_URL=https://api.openai.com/v1
EMBEDDING_API_KEY=sk-xxxx
EMBEDDING_API_MODEL=text-embedding-3-small
```

> 任意 OpenAI 兼容端点（如国内大模型网关）均可，只要提供 `/embeddings` 接口、返回 1536 维向量。

## 3. 启动前端

```bash
cd frontend
npm install
npm run dev
```

打开 http://localhost:5173 ：

1. 进入「文档管理」→ 上传 `docs/seed/vue3-notes.md`（或 `java-notes.md`）
2. 进入「问答」→ 提问，如「Vue3 的响应式原理是什么」，查看抽取式答案与引用来源

## 4. 部署（可选）

- 前端：`npm run build` → `dist/` 托管到 Vercel / Nginx
- 后端：打 jar 包或 Docker 镜像，配合环境变量注入数据库与 Embedding 配置
- 数据库：使用托管的 PostgreSQL + pgvector（如 Supabase / 云厂商）

## 验证清单

- [ ] `docker compose up -d` 后数据库可连接
- [ ] 后端启动无 Flyway 报错
- [ ] 上传 .md 后返回 `chunkCount > 0`
- [ ] 提问后返回 `sources` 且 `score` 合理
